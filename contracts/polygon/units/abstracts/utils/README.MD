> Initializable and configurable do the same thing. They are functions that can only be called once. Configurable is intended to be used on the proxy side. It should be used to set the initial implementation of the proxy, which will not have access control. The deployer should be the first one to call this function. Initializable should be called on the implementation side, which will initialize any required logic. This should also be called by the deployer as it may contain access control logic to grant ownership to the deployer. Both functions should be called one, and only once. If posssible these should be done by a deployer contract within the same transaction. The deployer should, deploy the proxy, configure the first implementation, and initialize the implementation on the newly configured implementation.
